# Ink Scenario Creator

このスキルは、`adventure-mcp` プロジェクトにおける Ink シナリオ作成を支援します。

## ワークフロー

1. **要件の理解と設計**
   ユーザーから提示されたアイデアやプロットをもとに、必要なフラグ（変数）やノット（場面）を設計します。
   adventure-mcp では、AIはゲームの状態を直接保持せず、すべての状態遷移やフラグ管理は Ink ファイル内で行う必要がある点に注意してください。
   プロジェクト特有の制約やベストプラクティスの詳細は [adventure_mcp_patterns.md](./references/adventure_mcp_patterns.md) を参照してください。

2. **シナリオの実装**
   要件に合わせて Ink の構文を使用し、シナリオを実装します。
   - 基本的な分岐、選択肢、簡単なフラグ管理が必要な場合は [ink_basics.md](./references/ink_basics.md) を参照してください。
   - より複雑なロジック（Weave、Thread、List、関数など）が必要な場合は [ink_advanced.md](./references/ink_advanced.md) を参照してください。

3. **ファイルの出力**
   実装したシナリオは、`engine/assets/` ディレクトリ配下に `.ink` 形式で保存します。

4. **コンパイルと検証**
   ファイル出力後、プロジェクトで用意されているコマンド `pnpm run build:ink` を実行し、生成した `.ink` ファイルが正常に JSON にコンパイルできるか確認してください。エラーが発生した場合は、標準出力の内容をもとに修正を行ってください。
