# Inkの基本構文リファレンス

Ink言語におけるシナリオ記述の基本的な文法と構造です。

## ノット (Knot) とステッチ (Stitch)

シナリオの基本的なブロック（節・段落）です。ノットの中にサブセクションとしてステッチを作ることができます。

```ink
=== room_entry ===
あなたは薄暗い部屋に入った。
-> look_around

= look_around
机の上にはランプが置かれている。
-> DONE
```

## ダイバート (Divert)

他のノットやステッチへの遷移を行います。

```ink
// ノットへの遷移
-> room_entry

// ステッチへの遷移（ノット名を省略できる場合）
-> look_around

// 他のノットのステッチへの遷移
-> room_entry.look_around

// シナリオの終了
-> DONE

// 呼び出し元のスレッド・分岐に戻る
->->
```

## 選択肢 (Choices)

プレイヤーに提示される選択肢です。一度選ぶと消えるものと、何度でも選べるものがあります。

```ink
* [一度だけ選べる選択肢]
    一度選ぶと二度と表示されません。
    -> DONE
+ [何度でも選べる選択肢]
    何度でも選ぶことができます。
    -> DONE

// 選択肢のテキストの表示制御
* [机を調べる] 机の上には手紙があった。
  // プレイヤーが選んだ後、「机を調べる」というテキストは表示されず、「机の上には手紙があった。」のみ表示される。
```

## 変数 (Variables)

状態を保持するための変数です。グローバル変数の宣言と、値の更新方法です。

```ink
// 変数の宣言 (シナリオの最上部付近に記述)
VAR has_key = false
VAR player_hp = 100
VAR player_name = "名無し"

=== room ===
// 変数の更新
~ has_key = true
~ player_hp = player_hp - 10

// 変数の値をテキストに埋め込む
あなたのHPは {player_hp} です。
-> DONE
```

## 条件分岐 (Conditionals)

変数の値や、特定のノットを訪れたかどうかでテキストや選択肢を出し分けます。

```ink
// テキスト内の条件分岐
{has_key:
    あなたは鍵を使って扉を開けた。
  - else:
    鍵がないため扉は開かない。
}

// 短い書き方
{has_key: 鍵を使って開けた。 | 鍵がない。}

// 選択肢の条件分岐
+ {has_key} [扉の鍵を開ける]
    -> open_door
+ {not has_key} [扉を調べる]
    -> check_door

// ノットに訪れた回数での分岐
{ room_entry > 1:
    またこの部屋に戻ってきた。
}
```

## 一時変数 (Temporary Variables)

特定のノット内でのみ有効な変数です。

```ink
=== combat ===
~ temp damage = 15
あなたは {damage} のダメージを受けた。
-> DONE
```
