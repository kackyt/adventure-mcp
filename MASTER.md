# MASTER.md - 開発マスターガイド

## 1. プロジェクト概要
AIエディタ上で生成AIと対話しながら謎解きや探索を進めるアドベンチャーゲームを提供するMCPサーバーです。シナリオの進行や状態管理は `Ink (inkle)` を用いて堅牢に行い、AIはプレイヤーとの対話（ゲームマスター役）に専念します。

## 2. 技術スタック

| カテゴリ | 採用技術 | 選定理由・役割 |
| --- | --- | --- |
| バックエンド | **Node.js + TypeScript** | MCPの実装しやすさ、SDKサポートの手厚さ。`inkjs`を利用するため。 |
| シナリオエンジン | **Ink (inkle)** | シナリオの分岐、アイテム/フラグの管理、デッドエンドの静的解析を行うため。 |
| データベース | なし（ファイル保存） | プレイヤーの進行状況(セーブデータ)の保存はJSONファイル等で十分とする。 |
| インフラ | なし | AIエディタ(Antigravity等)からローカルプロセスとして起動する。 |
| フロントエンド | なし | ユーザーUIはAIエディタのチャット画面を使用する。デバッグ用にCLIを用意する。 |

## 3. 開発ルール

### 言語・フォーマット
- ソースコードはTypeScriptで記述する。
- パッケージマネージャーはpnpmを使用する。

### MCPサーバーの責務
- **状態管理の秘匿**: AIに対して「ゲーム全体のシナリオ」や「未解明のフラグ」を公開しない。現在プレイヤーがいる場所、見えているもの、実行可能なアクションのみを提供する。
- **堅牢な状態遷移**: AIは「行動」をMCPツールとして呼び出すのみであり、実際のフラグ処理やアイテム付与の判定は必ずバックエンド(InkJS)側で行う。

### シナリオ開発 (Ink)
- `.ink` ファイルでシナリオを記述し、コンパイラ(`inklecate`)でエラーや到達不可能なノードがないかをチェックする。
- 最初に「1部屋からの脱出」程度のPoCで動作検証を行う。
