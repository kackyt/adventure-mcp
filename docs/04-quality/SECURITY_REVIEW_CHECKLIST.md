# セキュリティレビューチェックリスト

PRレビューおよびリリース前レビューで使用する。該当しない項目は `N/A` と理由を記載する。自動化済みの項目はCIログへのリンクで代替可。

## 認証・認可

- [ ] **認証**: パスワードハッシュ、セッション/JWT の有効期限・失効、多要素（要件がある場合）が設計どおりか
- [ ] **認可**: リソース単位のアクセス制御（IDOR 防止）、最小権限、管理操作の保護が一貫しているか
- [ ] **JWT/OAuth**（利用時）: 署名・`aud`/`iss` 検証、リフレッシュトークンの保管、CSRF と組み合わせた場合の方針が妥当か
- [ ] **機密情報**: シークレットがコード・ログ・エラー応答に含まれないか

## 入力検証

- [ ] **SQLインジェクション**: パラメータ化クエリ/ORMの正しい利用、動的SQLの有無と隔離
- [ ] **XSS**: 出力エスケープ、CSP・テンプレートエンジンの安全な利用（HTML挿入箇所の洗い出し）
- [ ] **インジェクション全般**: コマンド、LDAP、パス、テンプレート等の危険API利用の有無
- [ ] **ファイルアップロード**: 拡張子/MIME/サイズ制限、保存先の公開範囲、実行可能ファイルの拒否

## データ保護

- [ ] **転送中**: HTTPS の強制、適切な TLS 設定（中間者・ダウングレード）
- [ ] **保存時**: 必要なフィールドの暗号化、鍵管理（KMS/環境変数の取り扱い）
- [ ] **PII**: 収集最小化、保持期間、マスキング、ログ・ダンプへの漏えい防止、越境転送の要否

## 依存関係・サプライチェーン

- [ ] **脆弱性スキャン**: `npm audit` / OSV / Snyk 等の結果を確認し、許容できないFindingに対応したか
- [ ] **依存の信頼性**: 新規依存の必要性、メンテ状況、サプライチェーンリスクの判断記録（[DECISIONS.md](../06-reference/DECISIONS.md) またはチームの設計判断ログ）

## API・Web 全般

- [ ] **レート制限・DoS**: 認証エンドポイントや高コストAPIの保護
- [ ] **CORS・CSRF**: ブラウザ経由の操作で不適切な緩和になっていないか
- [ ] **エラー応答**: スタックトレースや内部構造がユーザー向けに露出しないか

## 運用

- [ ] **監査ログ**: セキュリティ関連操作の記録要件を満たすか
- [ ] **インシデント**: 新規リスクが [RISKS.md](../07-project-management/RISKS.md) またはチームのリスク台帳に反映されているか（該当時）
