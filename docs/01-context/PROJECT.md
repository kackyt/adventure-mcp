## 2. プロジェクトビジョン

### ミッションステートメント
生成AIとの対話と謎解きを融合した新しい体験を生むアドベンチャーゲーム

### ビジネス価値
- アドベンチャーゲームというジャンルに生成AIとの対話という新しい体験を生み出す
- 探索や戦闘の最中に、AIと対話しながらヒントをつかんで謎を解く体験の提供

### 成功指標
- ダウンロード数: 1000 (6ヶ月後)
- いいね数: 100 (6ヶ月後)
- SNSで話題になる: 10 (6ヶ月後)

## 4. 要件定義

### 機能要件

#### コア機能
- [x] アドベンチャーゲームを実行するロジック（Ink形式でシナリオを分岐させて進行）
- [x] ゲームロジックのMCP実装（生成AIがゲームロジックを参照できるようにする）
- [x] 生成AIと対話してゲームを進行させるAgent Skill（AIエディタ上で遊べるようにする）

#### 拡張機能
- [ ] 生成AIによるシナリオ自動生成

### 非機能要件
- 対象外：Webアプリ、モバイルアプリ対応（AIエディタのみ）
- 対象外：多言語対応（日本語のみ）

## 5. スコープ定義

### 含まれるもの
- MCPサーバー実装
- Ink言語によるシナリオロジック実装
- AIエディタでの動作を前提としたCLI/Agent Skill

### 含まれないもの
- グラフィックスUI (GUI)
- 自前のWebフロントエンドやスマホアプリ

## 6. リスクと前提条件

### 主要リスク
| リスク | 影響度 | 発生確率 | 対策 |
| --- | --- | --- | --- |
| ゲームが破綻する・動かない | 高 | 高 | AIの幻覚を防ぐため、状態管理はInkエンジン側に一任する |
| シナリオが面白くない | 高 | 高 | 最初に小規模なPoC（1部屋からの脱出）を作り面白さを検証する |
| シナリオを作る手間が莫大すぎ | 高 | 中 | Ink言語を活用し、構造化されたシナリオ作成を効率化。到達不可能性などはコンパイラで検知する |
| ナビゲートするAIがポンコツ | 高 | 中 | プロンプトで厳格にルールを縛り、現在見えている情報のみをAIに渡す |

### 前提条件
- ユーザーはAntigravityやCursorなどのMCPクライアントを所持していること
